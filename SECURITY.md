# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 0.8.x   | :white_check_mark: |
| < 0.8   | :x:                |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities by emailing **vral_parmar@pm.me** with:

- A description of the vulnerability and its potential impact
- Steps to reproduce or a proof-of-concept
- Any suggested fix if you have one

**What to expect:**

- Acknowledgement within **48 hours**
- A status update within **7 days** (accepted, declined, or needs more info)
- If accepted, a patched release will be published as soon as practical and you will be credited in the release notes (unless you prefer to remain anonymous)
- If declined, a clear explanation of why

Thank you for helping keep reptr secure.
